## 1. 프로젝트 목적

`yeoju_book_v1-수정요청.pdf`에 담긴 2025 여주 자전거 시티투어 사업 내용을 웹사이트로 전환한다. 사이트는 단순 홍보 페이지가 아니라, 일반 방문자에게는 투어의 매력과 이용 정보를 제공하고, 가이드/운영자에게는 운영 매뉴얼과 코스별 안내 자료를 제공하는 정적 웹사이트로 구축한다.

핵심 목표는 다음과 같다.

- 여주 자전거 시티투어와 `따르릉 여주 한글길`의 정체성을 명확하게 전달한다.
- PDF의 사업 소개, 코스 해설, 운영 가이드, FAQ, 비상연락망을 웹에 맞게 재구성한다.
- 네이버스토어 예약 링크와 카카오톡채널 문의 링크로 사용자를 연결한다.
- 한/영 지원, 라이트/다크 모드, PC/모바일 반응형을 제공한다.
- 검색포털 노출을 위해 SEO 구조를 처음부터 고려한다.
- 구현 전에 디자인 시스템을 먼저 구축하고, 애니메이션을 적극 활용한 경험형 사이트로 만든다.

## 2. 현재까지 확정된 방향

### 2.1 사이트 성격

사이트는 정적 페이지 중심으로 구축한다. 예약 처리, 결제, 문의 응답은 자체 서버에서 처리하지 않고 외부 서비스로 연결한다.

- 예약: 네이버스토어 링크로 이동
- 문의: 카카오톡채널 링크로 이동
- 검색 노출: SEO 메타 정보, 사이트맵, robots.txt, 네이버 서치어드바이저, 구글 서치콘솔 등록 고려
- 호스팅: 구매 도메인 + GitHub Pages 우선 검토

예약 링크와 문의 링크의 실제 URL은 추후 입력한다. 구현 시에는 링크 값을 설정 파일이나 콘텐츠 데이터에 분리해 두고, 실제 URL이 정해지면 코드 구조 변경 없이 교체할 수 있게 한다.

### 2.2 호스팅 판단

현재 요구사항에는 서버가 필요한 기능이 없다. 예약과 문의가 외부 링크로 분리되므로 GitHub Pages가 적합하다.

GitHub Pages 사용 조건:

- 정적 HTML/CSS/JavaScript 배포
- 커스텀 도메인 연결
- HTTPS 적용
- 빌드 결과물을 GitHub Actions 또는 Pages 배포 설정으로 게시

운영자/가이드 자료는 공개 메뉴에 노출하지 않고, 직접 URL을 아는 사람만 접근할 수 있는 숨김 URL 방식으로 제공한다. 이 방식은 보안 기능이 아니라 정보 구조상 비노출 처리이며, 서버 기반 로그인 보호는 제공하지 않는다.

### 2.3 SEO 방향

사용자가 말한 `SSO`는 로그인 SSO가 아니라 검색포털 노출을 의미하는 `SEO`로 정정되었다.

SEO 대상 키워드 예시는 다음과 같다.

- 여주 자전거 시티투어
- 따르릉 여주 한글길
- 여주 전기자전거 투어
- 남한강 자전거길
- 여주 한글길 코스
- 세종대왕릉 자전거 투어
- 여주 여행
- 여주 가이드 투어

SEO 기본 산출물:

- 페이지별 `title`, `description`
- Open Graph 이미지와 공유 문구
- `sitemap.xml`
- `robots.txt`
- 구조화된 heading 계층
- 한국어/영어 언어 속성
- 검색포털 등록용 기본 설명문

### 2.4 도메인 네이밍 방향

도메인은 브랜드명과 검색성을 함께 고려하는 방향으로 정한다.

우선 후보군:

- `yeojubiketour.kr`
- `yeojubiketour.com`
- `yeojuhangulroad.kr`
- `yeojuhangulroad.com`

가장 무난한 1순위는 `yeojubiketour.kr`이다. 검색 키워드인 `Yeoju`, `bike`, `tour`가 모두 들어가고, 외국인 방문자에게도 의미가 비교적 명확하다.

`yeojuhangulroad.kr`은 `한글길`의 브랜드 정체성을 더 잘 살리는 후보로 둔다. 다만 검색 유입 관점에서는 `bike tour`가 들어간 도메인보다 직관성이 낮을 수 있다.

실제 도메인 구매 전에는 등록 가능 여부, 가격, 보유자 여부, 유사 도메인 혼동 가능성을 확인한다.

예상 섹션:

1. 운영자 자료 홈

운영자 자료는 공개 내비게이션에 노출하지 않는다. `/guide` 같은 숨김 URL로 접근하는 정적 페이지 묶음으로 구성하며, 가이드와 운영자가 현장에서 빠르게 찾아볼 수 있도록 목차, 검색성 높은 섹션 제목, 체크리스트형 UI를 사용한다.

운영자 자료에서는 세종편과 오학편을 공개 사이트보다 더 자세히 분리한다. 각 코스별 운영 노트, 권장속도, PAS 단계, 위험 구간, 정차 지점, 해설 포인트를 현장용 체크리스트와 타임라인으로 제공한다.

## 5. 디자인 시스템 방향

디자인 시스템을 사이트 구현보다 먼저 구축한다.

### 5.1 디자인 원칙

- 여주 지역성, 남한강 자전거길, 한글 자음 스토리텔링을 시각적 중심으로 삼는다.
- 관광 홍보 사이트지만 운영자 자료까지 포함하므로 과하게 장식적인 랜딩 페이지가 아니라 정보 탐색성이 좋은 구조로 만든다.
- 애니메이션은 많이 활용하되, 콘텐츠 가독성과 성능을 해치지 않는다.
- 라이트/다크 모드를 모두 고려해 색상 토큰을 설계한다.
- 모바일에서도 코스 정보, 안전 정보, FAQ를 빠르게 찾을 수 있어야 한다.

### 5.2 디자인 톤

디자인 톤은 `활동적인 자전거 어드벤처형`과 `문화 스토리텔링형`을 혼합한다.

첫 화면과 주요 전환부에서는 라이딩, 코스, 속도감, 이동감을 강조한다. 스크롤 진행에 따라 코스 라인이 그려지고, 자전거 주행의 리듬이 느껴지는 애니메이션을 사용한다.

본문 섹션에서는 한글 자음, 세종, 여주 역사/문화 해설을 중심으로 스토리텔링을 강화한다. 장소별 자음 마커와 코스 내러티브를 연결해 `따르릉 여주 한글길`만의 차별성을 보여준다.

전체 인상은 역동적이지만 과격하지 않게 유지한다. 관광객에게는 참여 욕구를 주고, 운영자에게는 정보를 빠르게 찾을 수 있는 신뢰감 있는 구조를 제공한다.

### 5.3 비주얼 자산 방향

비주얼은 실제 사진과 생성 이미지/일러스트를 섞는 혼합형으로 확정한다.

실제 사진은 여주, 남한강, 자전거길, 주요 명소, 투어 현장 분위기를 보여주는 영역에 사용한다. 특히 Hero, 코스 소개, 명소 해설, 오시는 길 섹션은 실제 장소감과 신뢰도가 중요하므로 사진 중심으로 구성한다.

생성 이미지, 일러스트, 모션 그래픽은 한글 자음 스토리텔링, 코스 라인, 자음 마커, 안전 안내, 운영자/가이드 자료에 사용한다. 실제 사진만으로 설명하기 어려운 흐름, 절차, 체크리스트, 수신호, 장비 사용법은 그래픽 기반으로 표현한다.

사진과 그래픽이 따로 노는 느낌을 줄이기 위해 동일한 색상 토큰, 라인 두께, 코너 반경, 자음 마커 스타일을 반복 사용한다.

### 5.4 토큰 초안

색상 토큰은 다음 의미 단위로 구성한다.

- `background`: 전체 배경
- `surface`: 카드/패널 배경
- `surface-elevated`: 강조 패널 배경
- `text`: 본문 텍스트
- `text-muted`: 보조 텍스트
- `border`: 구분선
- `accent-river`: 남한강/자전거길 계열 강조색
- `accent-rice`: 여주 쌀/들판 계열 강조색
- `accent-hangul`: 한글길/문화 계열 강조색
- `accent-safety`: 안전 안내 계열 강조색
- `danger`: 비상/주의
- `success`: 정상/완료

### 5.6 애니메이션 방향

애니메이션은 자전거 주행과 한글 자음 스토리텔링의 리듬을 살리는 방식으로 사용한다.

적용 후보:

- 스크롤에 따라 코스 라인이 그려지는 효과
- 코스 포인트가 순차적으로 활성화되는 효과
- 한글 자음 마커가 등장하는 전환
- Hero에서 남한강 자전거길의 이동감을 주는 패럴랙스
- FAQ/운영자 체크리스트의 부드러운 펼침/접힘
- 예약/문의 CTA의 시선 유도 모션
- 라이트/다크 모드 전환 시 부드러운 색상 전환

접근성 기준:

- `prefers-reduced-motion` 사용자의 경우 과한 모션을 줄인다.
- 정보 이해에 필요한 내용을 애니메이션에만 의존하지 않는다.
- 모바일에서 성능 저하가 큰 모션은 제한한다.

### 5.7 디자인 시스템 상세안

디자인 시스템은 `Yeoju Ride System`이라는 작업명으로 정리한다. 핵심은 강변 라이딩의 활동성, 한글 자음 스토리텔링의 문화성, 운영 자료의 실무성을 하나의 시각 언어로 묶는 것이다.

#### 5.7.1 색상 토큰

라이트 모드는 맑은 강변과 낮 시간 라이딩을 기준으로 한다. 다크 모드는 야간 운영자 자료 확인과 모바일 현장 사용을 고려해 대비를 높인다.

| 토큰 | 라이트 | 다크 | 용도 |
| --- | --- | --- | --- |
| `--color-bg` | `#F7FAF8` | `#0D1412` | 전체 배경 |
| `--color-surface` | `#FFFFFF` | `#14201D` | 카드, 패널 |
| `--color-surface-alt` | `#EDF5F1` | `#1B2A26` | 섹션 배경, 보조 패널 |
| `--color-text` | `#13201B` | `#EFF7F3` | 기본 텍스트 |
| `--color-text-muted` | `#5E6F68` | `#A9BBB4` | 보조 설명 |
| `--color-border` | `#D7E4DE` | `#2C403A` | 구분선 |
| `--color-river` | `#167C80` | `#4FC3C7` | 남한강, 코스 라인 |
| `--color-ride` | `#F26B3A` | `#FF936B` | CTA, 라이딩 에너지 |
| `--color-hangul` | `#3046A6` | `#8EA2FF` | 한글 자음, 문화 스토리텔링 |
| `--color-rice` | `#C99A2E` | `#E7C66A` | 여주 쌀, 들판, 지역성 |
| `--color-safety` | `#2F8F5B` | `#67D49A` | 안전, 정상 상태 |
| `--color-danger` | `#C94132` | `#FF7A70` | 비상, 주의 |

색상 사용 규칙:

- 공개 사이트의 주 강조색은 `river`와 `ride`를 사용한다.
- 한글 자음 마커와 코스 키워드는 `hangul`을 사용한다.
- 농업, 쌀, 들판, 여주 지역성 강조에는 `rice`를 제한적으로 사용한다.
- 운영자/가이드 자료의 체크리스트와 안전 정보는 `safety`와 `danger`를 명확히 분리한다.
- 다크 모드에서도 CTA는 `ride` 계열을 유지해 예약/문의 버튼의 인지도를 떨어뜨리지 않는다.

#### 5.7.2 타이포그래피

기본 글꼴은 한국어와 영어 모두 읽기 쉬운 산세리프 계열을 사용한다.

- 기본: `Pretendard`, `Noto Sans KR`, `system-ui`, `sans-serif`
- 숫자/거리/시간: `Inter`, `Pretendard`, `system-ui`, `sans-serif`
- 한글 자음 마커: 기본 글꼴을 사용하되 굵기와 원형 배경으로 브랜드성을 만든다.

타입 스케일:

| 이름 | Desktop | Mobile | 용도 |
| --- | --- | --- | --- |
| `display` | `64px / 1.05 / 800` | `40px / 1.1 / 800` | Hero 핵심 문구 |
| `h1` | `48px / 1.12 / 800` | `34px / 1.15 / 800` | 페이지 제목 |
| `h2` | `36px / 1.18 / 750` | `28px / 1.2 / 750` | 주요 섹션 제목 |
| `h3` | `24px / 1.3 / 700` | `21px / 1.32 / 700` | 카드/패널 제목 |
| `body` | `17px / 1.75 / 400` | `16px / 1.7 / 400` | 본문 |
| `body-strong` | `18px / 1.65 / 650` | `16px / 1.65 / 650` | 강조 본문 |
| `caption` | `14px / 1.5 / 500` | `13px / 1.45 / 500` | 보조 설명 |
| `control` | `15px / 1 / 700` | `14px / 1 / 700` | 버튼, 탭, 토글 |

타입 규칙:

- Hero 문구는 짧고 힘 있게 유지한다.
- 운영자 자료는 본문 가독성을 우선해 `body` line-height를 넉넉하게 둔다.
- 버튼과 탭은 브라우저 기본 크기를 쓰지 않고 `control` 스타일을 명시한다.
- 긴 한국어 문장은 모바일에서 2-4줄로 자연스럽게 줄바꿈되도록 최대 너비를 제한한다.

#### 5.7.3 레이아웃과 공간

전체 레이아웃은 넓은 강변 경로처럼 수평 흐름을 가지되, 모바일에서는 세로 타임라인 중심으로 전환한다.

브레이크포인트:

- `mobile`: 0-767px
- `tablet`: 768-1023px
- `desktop`: 1024-1439px
- `wide`: 1440px 이상

공간 토큰:

| 토큰 | 값 | 용도 |
| --- | --- | --- |
| `--space-1` | `4px` | 미세 간격 |
| `--space-2` | `8px` | 아이콘/텍스트 간격 |
| `--space-3` | `12px` | 작은 컨트롤 |
| `--space-4` | `16px` | 카드 내부 기본 |
| `--space-5` | `24px` | 섹션 내부 묶음 |
| `--space-6` | `32px` | 카드 간격 |
| `--space-7` | `48px` | 섹션 블록 |
| `--space-8` | `72px` | 대형 섹션 |
| `--space-9` | `112px` | Hero/주요 전환 |

레이아웃 규칙:

- 본문 컨테이너 최대 너비는 `1180px`로 둔다.
- 운영자 자료의 읽기 영역은 `860px` 안팎으로 제한해 긴 문서를 읽기 쉽게 한다.
- Hero는 첫 화면에서 브랜드와 투어 성격이 바로 보이되, 다음 섹션의 일부가 보이도록 높이를 조정한다.
- 공개 사이트는 섹션을 카드로 감싸기보다 full-width band와 개별 컴포넌트 조합으로 구성한다.
- 카드 중첩은 금지한다. 반복 항목, FAQ, 비상 연락처, 코스 요약처럼 명확히 구분되는 단위에만 카드 스타일을 쓴다.

#### 5.7.4 형태, 선, 아이콘

형태 규칙:

- 기본 radius는 `8px` 이하로 제한한다.
- Hero 이미지나 큰 미디어 프레임은 `12px`까지 허용한다.
- 자음 마커는 원형 또는 살짝 눌린 원형으로 통일한다.
- 코스 라인은 `3px` 선을 기본으로 하고, 활성 구간은 `5px`까지 두껍게 표시한다.

아이콘 규칙:

- 일반 UI 아이콘은 `lucide` 스타일의 얇은 선형 아이콘을 기준으로 한다.
- 자전거, 지도, 안전, 문의, 예약 아이콘은 동일한 stroke width를 유지한다.
- 한글 자음은 아이콘처럼 사용하되 텍스트 접근성을 유지한다.
- 위험/주의 아이콘은 색상만으로 구분하지 않고 형태와 라벨을 함께 제공한다.

#### 5.7.5 컴포넌트 상세

Header:

- 좌측에는 사이트명 또는 간결한 로고 텍스트를 둔다.
- 공개 메뉴는 `투어 소개`, `코스`, `안전`, `FAQ`, `오시는 길` 중심으로 제한한다.
- `/guide`는 공개 메뉴에 노출하지 않는다.
- 우측에는 언어 전환, 다크모드 토글, 예약 CTA를 배치한다.

Hero:

- 실제 여주/남한강/자전거길 사진을 배경 또는 주 미디어로 사용한다.
- 텍스트는 카드 안에 넣지 않고 이미지와 열린 배경 위에 배치한다.
- CTA는 `예약하기`, `카카오톡 문의` 2개를 기본으로 한다.
- Hero 안에서 과한 설명 문구를 늘리지 않고, 첫 화면에서 투어명과 성격이 바로 보이게 한다.

Course Tabs:

- 세종편과 오학편을 같은 위계의 탭으로 제공한다.
- 탭 전환 시 총 거리, 예상 시간, 난이도, 주요 방문지, 코스 타임라인이 함께 전환된다.
- 코스 포인트는 `ㄱ`, `ㄴ`, `ㄷ` 같은 자음 마커와 장소명을 함께 보여준다.
- 활성 코스 라인은 스크롤 또는 탭 전환에 맞춰 그려지는 모션을 사용한다.

Course Timeline:

- Desktop에서는 지도형 또는 수평 경로형 시각화를 우선한다.
- Mobile에서는 세로 타임라인으로 전환한다.
- 각 포인트에는 장소명, 해설 한 줄, 안전/운영 메모 진입점을 둔다.

Guide Checklist:

- 운영자 자료에서 사용한다.
- 출발 전, 주행 중, 정차 지점, 비상 상황으로 묶는다.
- 체크박스는 실제 저장 기능 없이도 시각적 체크 상태를 제공할 수 있다. 저장 기능은 정적 사이트 범위를 넘기므로 구현 계획에서 별도 판단한다.

FAQ Accordion:

- 공개 FAQ와 운영자 FAQ를 분리한다.
- 질문은 검색 키워드가 되므로 실제 사용자 질문 문장 형태로 작성한다.
- 펼침/접힘 모션은 180-240ms 안에서 처리한다.

Emergency Contact:

- 운영자 자료와 공개 안내 모두에서 접근 가능하게 한다.
- `112`, `119`, 병원, 보건소, 택시를 시각적으로 구분한다.
- 모바일에서는 전화 링크로 전환할 수 있게 설계한다.

External CTA:

- 예약과 문의는 항상 외부 링크임을 명확하게 표시한다.
- 실제 링크가 입력되기 전에는 비활성 처리하거나 `추후 입력` 상태로 둔다.
- 구현 시 링크 텍스트와 URL을 콘텐츠 데이터에서 관리한다.

#### 5.7.6 모션 시스템 상세

모션은 많이 사용하되 세 가지 범주로 제한해 일관성을 유지한다.

| 범주 | 속도 | easing | 용도 |
| --- | --- | --- | --- |
| `quick` | `120-180ms` | `ease-out` | 버튼 hover, 탭 선택 |
| `standard` | `220-320ms` | `cubic-bezier(0.2, 0.8, 0.2, 1)` | 카드 등장, FAQ 열림 |
| `story` | `600-1200ms` | `cubic-bezier(0.16, 1, 0.3, 1)` | 코스 라인, Hero 패럴랙스, 자음 등장 |

주요 모션:

- Hero 진입 시 사진은 천천히 확대되고, 코스 라인 또는 자음 마커가 순차 등장한다.
- 코스 섹션에서는 탭 전환 시 코스 라인이 다시 그려진다.
- 스크롤 진입 시 섹션 제목, 요약 수치, 코스 포인트가 순차적으로 올라온다.
- 운영자 자료에서는 과한 패럴랙스보다 접힘/펼침, 체크, 강조 표시 같은 실무형 모션을 사용한다.

모션 제한:

- 텍스트가 움직이는 동안 읽기 어려워지면 안 된다.
- 모바일에서 `story` 모션은 한 화면에 1개만 실행한다.
- `prefers-reduced-motion: reduce`에서는 라인 그리기와 패럴랙스를 정적 표시로 대체한다.

#### 5.7.7 공개 사이트와 운영자 자료의 UI 차이

공개 사이트:

- 감성적이고 활동적인 첫인상
- 사진과 모션 중심
- 예약/문의 전환 강조
- 코스의 매력과 선택을 쉽게 이해시키는 구성

운영자 자료:

- 빠른 탐색과 현장 사용성 우선
- 목차, 체크리스트, 경고 박스, 연락처 중심
- 사진보다 도식, 절차, 타임라인 중심
- 모바일에서 한 손으로 읽기 쉬운 간격과 큰 터치 영역

두 영역은 같은 색상과 타입 토큰을 공유하되, 공개 사이트는 `ride`와 `river`를 더 많이 쓰고 운영자 자료는 `safety`, `danger`, `surface-alt`를 더 많이 사용한다.

#### 5.7.8 접근성과 반응형 기준

- 텍스트와 배경 대비는 WCAG AA 수준을 목표로 한다.
- CTA 버튼은 최소 `44px` 높이를 유지한다.
- 모바일에서 탭, 토글, FAQ 버튼은 엄지 조작을 고려해 충분한 간격을 둔다.
- 색상만으로 상태를 전달하지 않는다.
- 실제 사진 위 텍스트는 반드시 충분한 대비를 확보한다.
- 애니메이션이 꺼져도 모든 정보와 CTA는 동일하게 접근 가능해야 한다.
